Resumen General de la Versión

Esta versión se enfoca en resolver inconsistencias financieras y errores de flujo en las integraciones de delivery. Se corrige un error en el cálculo de precios de subítems en PedidosYa que generaba cobros duplicados, se optimiza la interfaz de mapeo eliminando parámetros obsoletos y se soluciona un bug en el manejo de webhooks de UberEats que afectaba el control de flujo entre notificaciones y fallas.

Análisis Técnico Detallado

api/src/platforms/interfaces/pedidosYa.js

Optimización de Firma de Función y Limpieza

Se removió el parámetro redundante tenant en la llamada a detailsMapper dentro del procesamiento de órdenes. Este cambio mejora la legibilidad y mantiene la consistencia con la definición actual del mapeador.

Código Previo
// news order processing
news.order.details = detailsMapper(data.order, tenant);
Código Nuevo
// news order processing
news.order.details = detailsMapper(data.order); // Cleanup: tenant parameter removed

Corrección en Cálculo de Precio Acumulado

Se detectó que en detailsMapperParseOther se multiplicaba el precio del subítem por su cantidad de forma redundante, ya que la lógica superior ya contempla la cantidad por separado. Esto causaba una sobreestimación en el acumPrice.

Código Previo
// detailsMapperParseOther
acumPrice += parseFloat(option.price * option.quantity);
Código Nuevo
// detailsMapperParseOther
acumPrice += parseFloat(option.price); // Corrected: quantity multiplication removed to avoid duplication
