Resumen General de la Versión

Esta actualización representa una refactorización mayor enfocada en la eficiencia. Se centraliza la inicialización de plataformas mediante un sistema de caché a nivel de módulo, eliminando cientos de consultas redundantes a la base de datos por cada notificación. Además, se simplifica la gestión de datos internos y se robustece el manejo de errores en integraciones críticas como SML.

Análisis Técnico Detallado

api/src/utils/initPlatform.js (Nuevo Archivo)

Análisis General

Se introduce un componente centralizado para la instanciación de plataformas. Implementa un patrón de caché utilizando un Map para almacenar promesas, garantizando que cada plataforma se inicialice una sola vez por ciclo de vida.

api/src/controllers/ (mercadoPago.js, peya.js, uberEats.js...)

Análisis General

Se migran los controladores para utilizar el nuevo sistema de inicialización compartido. Esto elimina la lógica local redundante y las consultas directas a PlatformModel.

Cambios Detallados

Implementación de Caché de Instancia

Explicación: Se reemplaza la creación ad-hoc de plataformas por el uso de _platformInstancePromise. Si la instancia ya existe en memoria, se reutiliza inmediatamente.

Código Antes: 

const initPlatform = (internalCode, uuid) => {
  const platformFactory = new PlatformFactory();
  return platformFactory.createPlatform(...);
}
// ... uso dentro de la función ...
const platform = await initPlatform(8, req.uuid);
Código Después: 

import initPlatform from '../utils/initPlatform';
let _platformInstancePromise = null;

const mercadoPagoNotification = async (req, res) => {
    if (!_platformInstancePromise) 
      _platformInstancePromise = await initPlatform(8, req.uuid);
    const platform = _platformInstancePromise;
    // ...
}

api/src/platforms/management/platform/mercadoPago.js

Análisis General

Optimización del acceso a propiedades. La clase ahora aprovecha su estado interno heredado para obtener credenciales, en lugar de consultar la base de datos repetidamente.

Cambios Detallados

Uso de Estado Interno (this._platform)

Explicación: Se eliminan las llamadas a PlatformModel.findOne. Las URIs y credenciales se extraen directamente de this._platform.credentials.

Código Antes: 

const mercadoPago = await PlatformModel.findOne({ name: 'MercadoPago' });
const urlTokenMercadoPagoCloud = `${mercadoPago.credentials.data.mercadoPagoGetToken_uri}${user_id}`;
Código Después: 

// constructor
this.mercadoPagoGetToken_uri = this._platform.credentials.data.mercadoPagoGetToken_uri;

// method
const urlTokenMercadoPagoCloud = `${this.mercadoPagoGetToken_uri}${user_id}`;
